sf-groups-toggler

Toggle Symfony serialization groups on your entity properties — right from the command line.

View on GitHub

What is it?

sf-groups-toggler is a CLI tool that reads a PHP entity file and adds or removes Symfony serialization #[Groups([...])] attributes on a chosen property — all while preserving your original code formatting.

Installation

git clone https://github.com/aharabara/sf-groups-toggler.git
cd sf-groups-toggler
composer install

Usage

php ./bin/parser <path/to/entity.php> <propertyName> <group1,group2,...>

Groups must be comma-separated with no spaces.

Examples

Add groups read, edit, list to the slug property:

php ./bin/parser ./examples/entity.php slug read,edit,list

Toggle (remove) the edit group while keeping the rest:

php ./bin/parser ./examples/entity.php slug edit

How it works

  1. The tool parses the target PHP file using nikic/php-parser.
  2. It locates the property you specify.
  3. If the property already has a #[Groups([...])] attribute:
    • Groups that are already present are removed (toggle off).
    • Groups that are not yet present are added (toggle on).
  4. The file is rewritten using format-preserving pretty-printing — your original style is kept intact.

Example entity

#[ORM\Column(length: 255)]
#[Groups(['delete', 'create', 'read'])]
private ?string $slug = null;

After running php ./bin/parser entity.php slug edit:

#[ORM\Column(length: 255)]
#[Groups(['delete', 'create', 'read', 'edit'])]
private ?string $slug = null;
